- `sync_identity` — pull users / groups / memberships into ZTNA Teams.

- `list_entitlements` — pull current permissions for an access check-up.
- `get_access_log` — pull sign-in / permission-change audit events into the audit pipeline.
- `sso_federation` — broker SAML / OIDC through Keycloak.

### 3.2 LangGraph DAG runtime (Phase 8 — closed)

- **Why.** The Phase 8 `WorkflowExecutor` walked steps linearly; branch fan-out (a single step routing into N parallel branches that join later) was the last documented Phase 8 gap.
- **Shipped (PR #22).** `WorkflowStepDefinition.Next` (fan-out array of step indices) and `WorkflowStepDefinition.Join` (fan-in array of predecessor step indices) extend `access_workflows.steps`. `WorkflowExecutor.Execute` detects DAG-shaped workflows (any step with `Next` or `Join`) and delegates to `executeDAG`, which validates against self-loops, out-of-range references, and cycles (Kahn's algorithm) before walking the DAG in topological order with goroutine-parallel branches; per-step outcomes (decision + error) are still recorded into `access_workflow_step_history` so `ListFailedSteps` keeps working as the DLQ view. Linear pipelines (no `Next` / `Join`) keep the legacy execution path. Tests cover linear backwards-compat, fan-out, fan-out+join, failure-in-one-branch, cycle rejection, out-of-range rejection, and step-error propagation.
- **Out of scope.** Re-architecting the four step bodies — they continue to return `approve` / `deny` / `escalate` / `pending`.
